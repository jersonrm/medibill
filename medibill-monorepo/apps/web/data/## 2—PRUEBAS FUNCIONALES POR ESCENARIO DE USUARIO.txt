## 2.5 — Hallazgos CRITICAL

### CRITICAL 1: RBAC no verificado en server actions de facturas
- **Archivo:** [facturas.ts](medibill-monorepo/apps/web/app/actions/facturas.ts#L148)
- **Problema:** `aprobarFactura()` usa `getContextoOrg()` para obtener `ctx.orgId` pero NUNCA llama `verificarPermisoOError(ctx.rol, "aprobar_factura")`. Un `facturador` podría aprobar facturas directamente llamando la server action, aunque la UI no le muestre el botón.
- **Impacto:** Bypass de RBAC — roles `facturador` y `auditor` pueden ejecutar acciones que no deberían.
- **Afecta también a:** `anularFactura()`, `editarFacturaBorrador()`, `crearFacturaBorrador()`.
- **Fix:** Agregar `verificarPermisoOError(ctx.rol, "aprobar_factura")` al inicio de cada action restringida.

### CRITICAL 2: RBAC no verificado en acuerdos
- **Archivo:** [acuerdos.ts](medibill-monorepo/apps/web/app/actions/acuerdos.ts#L63)
- **Problema:** `guardarAcuerdo()` y `eliminarAcuerdo()` no verifican `gestionar_acuerdos` permission. Doctor puede crear/eliminar acuerdos.
- **Fix:** Agregar `verificarPermisoOError(ctx.rol, "gestionar_acuerdos")`.

### CRITICAL 3: RBAC no verificado en clasificación IA
- **Archivo:** [clasificacion.ts](medibill-monorepo/apps/web/app/actions/clasificacion.ts)
- **Problema:** `clasificarTextoMedico()` verifica `verificarLimite` pero no verifica que el rol tenga `clasificar_ia` permission.
- **Fix:** Agregar check de permiso.
